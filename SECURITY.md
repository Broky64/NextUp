# Security Policy

## Supported Versions

Security fixes are currently applied to the latest `main` branch and the most recent GitHub release.

| Version | Supported |
| ------- | --------- |
| Latest release | Yes |
| `main` branch | Yes |
| Older releases | No |

## Reporting a Vulnerability

Please do **not** report security issues in public GitHub issues.

Use one of the following channels:

1. **Preferred**: GitHub "Report a vulnerability" (private security advisory).
2. If advisory flow is unavailable, open a private contact request through GitHub and reference that it is a security report for NextUp.

When reporting, include:

- A clear description of the issue
- Reproduction steps or proof of concept
- Impact assessment
- Suggested mitigation (if known)

## Response Process

- Initial triage target: within **5 business days**
- Status update cadence: at least every **7 business days** while active
- Fix timeline depends on severity and maintainers' availability

## Disclosure Policy

- Please allow time for triage and patching before public disclosure.
- After a fix is available, coordinated disclosure is encouraged.
